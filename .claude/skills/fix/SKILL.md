name: fix
description: 收集报错信息，压缩上下文，并使用 Agent 团队自动修复代码问题

---

## 使用方式

在 `/check` 发现错误后运行：

```
/fix
```

或者带上具体的错误信息：

```
/fix TypeScript 编译失败，错误：Cannot find module './utils'
```

## 执行流程

```
收集报错信息 ──► /compact <报错信息> ──► 启动修复 subagent ──► 修复代码 ──► 验证修复
```

## 详细步骤

### 1. 收集报错信息

从以下来源收集错误（如果用户未提供）：
- 最近一次的 `/check` 输出
- 终端中的编译/测试错误
- 用户输入的错误描述

### 2. 压缩上下文

运行：

```
/compact 需要修复的问题：
<收集到的报错信息>

请保留与修复这些错误相关的上下文。
```

### 3. 启动修复 Subagent

**类型**: `general-purpose`

**任务描述**:

基于压缩后的上下文和报错信息，修复代码中的问题：

1. **分析问题**: 理解错误的根本原因
2. **定位文件**: 找到需要修改的代码文件
3. **实施修复**:
   - 修复类型错误（TypeScript/Python）
   - 修复语法错误
   - 修复导入/路径问题
   - 修复 ESLint/ruff 规范问题
4. **验证修复**: 运行相关检查命令确认问题已解决

**工作目录**: `/Users/xtzn/Desktop/double-agent`

**输出要求**:
- 列出修复的文件和修改内容
- 说明每个修复的原因
- 最后验证结果：通过 ✅ 或仍需处理 ⚠️

### 4. 修复后验证

修复完成后，建议再次运行 `/check` 确认所有问题已解决。

## 与 /check 配合使用

典型工作流：

```
用户: 完成代码修改任务
Claude: (TaskCompleted hook 自动触发 /check)
/check: 发现前端类型错误和 Python 语法错误
用户: /fix
/fix: 收集错误 → /compact → agent修复 → 完成
用户: (可选) 再次运行 /check 验证
```

## 修复范围

| 错误类型 | 修复方式 |
|---------|---------|
| TS 类型错误 | 添加/修正类型注解 |
| TS 导入错误 | 修正路径或安装缺失依赖 |
| ESLint 问题 | 按规则修复代码风格 |
| Python 语法错误 | 修正语法 |
| mypy 类型错误 | 添加类型注解 |
| ruff 规范问题 | 按规则格式化代码 |
| 测试失败 | 分析失败原因并修复代码或测试 |

## 注意事项

- `/fix` 会调用 `/compact`，会压缩历史消息，如需保留完整历史请先导出
- 复杂问题可能需要多次 `/fix` 循环
- 如果修复涉及架构性改动，建议先进入 plan mode 规划
